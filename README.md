# 智能烟火检测系统

## 1. 项目简介

本项目旨在开发一个基于 RKNN3588 边缘计算平台的实时智能烟火检测系统。系统采用 Python 语言实现，结合 Streamlit 框架提供直观的 Web 交互界面，实现从 RTSP 视频流捕获、NPU 加速推理、检测结果解析到告警生成与历史回溯的完整闭环。

**核心目标：**
*   **实时性：** 低延迟地处理视频流并进行烟火检测。
*   **高效性：** 充分利用 RKNN3588 的 NPU 硬件加速能力。
*   **易用性：** 通过 Streamlit 提供友好的 Web 界面进行监控、配置和告警管理。
*   **可追溯性：** 记录所有告警事件及快照，便于后期审计和分析。

## 2. 系统架构

系统采用前后端分离的多线程架构，后端负责核心的视频处理和推理逻辑，前端负责用户交互和数据展示。

```
┌───────────────────┐       ┌───────────────────┐
│   RTSP 视频源     │       │   Streamlit Web   │
└─────────┬─────────┘       └─────────┬─────────┘
          │ RTSP 流                   │ WebSocket (文件模拟)
          │                           │
┌─────────▼─────────┐       ┌─────────▼─────────┐
│  后端服务 (server.py) │       │  前端应用 (app.py)  │
│ ┌─────────────────┐ │       │ ┌─────────────────┐ │
│ │ 抓帧线程 (FFmpeg) │─┬─────▶ │ 实时视频看板      │ │
│ ├─────────────────┤ │       ├─────────────────┤ │
│ │ 图像预处理      │ │       │ 告警与日志面板    │ │
│ │ NPU 推理 (RKNN) ├─┼─────▶ │ 系统控制与配置    │ │
│ ├─────────────────┤ │       └─────────────────┘ │
│ │ 结果后处理      │ │                           │
│ ├─────────────────┤ │                           │
│ │ 告警生成        │ │                           │
│ └─────────────────┘ │                           │
└─────────┬─────────┘                           │
          │ SQLite 数据库                       │
          │ 告警快照                            │
┌─────────▼─────────┐                           │
│   本地存储        │                           │
│ (alerts.db, snapshots/) │                           │
└───────────────────┘                           │
```

**核心组件：**
*   **RTSP 流捕获与解码：** 后端通过 OpenCV (模拟 FFmpeg) 捕获 RTSP 视频流，并进行关键帧解码。
*   **图像预处理与 NPU 推理：** 捕获到的帧经过 `letterbox` 预处理后，送入 RKNN 引擎进行目标检测。若 RKNN 环境不可用，则自动降级为模拟推理。
*   **检测结果解析与告警生成：** 推理结果经过后处理（如 NMS，本项目简化为直接使用检测结果），生成标注框，并根据置信度阈值判断是否触发告警。
*   **前后端通信：** 采用文件读写方式模拟 WebSocket 全双工通信，后端将处理后的视频帧和告警信息写入 `comm.json`，前端读取并更新界面。前端通过写入 `cmd.txt` 向后端发送控制指令。
*   **告警记录与快照：** 所有告警事件及带标注框的快照图片存储在本地 SQLite 数据库和 `snapshots/` 目录中。
*   **Streamlit Web 交互界面：** 提供实时视频监控、告警弹窗、历史告警查询、系统控制与配置等功能。

## 3. 核心功能

*   **实时视频监控：** 通过 Streamlit 界面实时展示来自 RTSP 流的视频画面。
*   **智能烟火检测：** 利用 RKNN 模型对视频帧进行烟火目标检测，并支持在 NPU 环境不可用时自动降级为模拟检测。
*   **告警实时推送：** 检测到烟火时，前端立即弹出模态告警弹窗，并更新实时告警面板。
*   **告警历史记录：** 所有告警事件（包括时间、摄像头、类型、置信度、快照路径等）存储在 SQLite 数据库中，支持前端查询和导出。
*   **告警快照保存：** 每次告警发生时，自动保存一张带标注框的快照图片。
*   **系统控制：** 前端提供“启动/停止系统”按钮，控制后端视频流处理和推理的启停。
*   **RTSP 配置管理：** 前端可实时修改 RTSP 地址、摄像头名称和置信度阈值，并保存到配置文件。
*   **连接测试：** 提供 RTSP 连接测试功能，验证配置的有效性。

## 4. 技术栈

*   **Python 3.8+**
*   **Streamlit：** 用于构建 Web 交互界面。
*   **OpenCV (cv2)：** 用于视频流捕获、图像处理和绘制。
*   **RKNNLite (rknnlite)：** 瑞芯微 NPU 推理库（可选，若环境不支持则降级）。
*   **SQLite3：** 用于本地告警数据存储。
*   **Numpy：** 科学计算库。
*   **Pandas：** 数据处理（用于历史告警展示）。

## 6. 安装与运行

### 6.1 环境准备

1.  **Python 环境：** 确保安装 Python 3.8 及以上版本。
2.  **依赖安装：**
    ```bash
    pip install streamlit opencv-python numpy pandas
    # 如果在 RKNN 平台上运行，且需要 NPU 加速，请安装 rknnlite
    # pip install rknn_toolkit_lite2-X.X.X-cp3X-cp3X-linux_aarch64.whl
    ```
3.  **FFmpeg：** 确保系统已安装 FFmpeg，OpenCV 依赖其进行视频流解码。
    ```bash
    sudo apt update
    sudo apt install ffmpeg
    ```
4.  **RKNN 模型：** 将 RKNN 模型文件 `det_helmet_person_n_260213.rknn` 放置在 `/zmn/streaminfer/evai_test/` 目录下。

### 6.2 启动系统

进入项目根目录 `/zmn/streaminfer/streamlit_demo`，执行启动脚本：

```bash
cd /zmn/streaminfer/streamlit_demo
./start.sh
```

脚本将同时启动后端服务和 Streamlit 前端应用。

### 6.3 访问界面

系统启动后，在浏览器中访问以下地址：

[http://127.0.0.1:8501](http://127.0.0.1:8501)

### 6.4 停止系统

进入项目根目录 `/zmn/streaminfer/streamlit_demo`，执行停止脚本：

```bash
cd /zmn/streaminfer/streamlit_demo
./stop.sh
```

## 7. 注意事项

*   **RKNN 环境：** 本项目后端尝试加载 RKNN 模型进行推理。如果运行环境没有正确安装 RKNN 驱动或模型加载失败，系统将自动降级为基于 HSV 颜色阈值的模拟检测，以保证系统功能演示的完整性。
*   **RTSP 流：** 默认配置使用虚拟 RTSP 流 `rtsp://127.0.0.1:8554/stream`。如果该流不可用，后端会自动生成模拟视频流。你可以在前端侧边栏的“RTSP 流配置”中修改为真实的 RTSP 地址。
*   **前后端通信：** 为简化部署和避免依赖问题，本项目的前后端通信通过读写 `comm.json` 和 `cmd.txt` 文件进行模拟，而非实际的 WebSocket 连接。在生产环境中，建议替换为真实的 WebSocket 实现以获得更佳性能和实时性。
*   **Streamlit `st.rerun()`：** 前端为了实现实时更新，使用了 `st.rerun()` 机制，这会导致页面频繁刷新。在实际应用中，可能需要更精细的优化来减少不必要的重绘。
*   **告警弹窗：** 告警弹窗功能依赖 Streamlit 1.34+ 版本。
